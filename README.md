# Clash Rules

这个仓库用于维护 Clash 分流规则、规则集和配置模板，主要面向妙妙屋环境适配。

仓库不包含可执行代码，核心内容是 Clash 规则文件、规则提供器和订阅转换用的配置模板。

## 主要用途

- 为妙妙屋提供可直接引用的 Clash 规则模板。
- 维护 AI、媒体、开发、Web3、游戏平台等服务的独立分流规则。
- 控制规则命中顺序，避免通用规则抢先命中具体服务。
- 提供 DNS 泄漏规避场景下使用的配置模板。

## 关键文件

| 路径 | 说明 |
| --- | --- |
| `dns_leak.yml` | 妙妙屋使用的 DNS 泄漏防护模板之一 |
| `no_dns_leak.yml` | 妙妙屋使用的无 DNS 泄漏模板之一 |
| `Clash/xiaobai.ini` | 当前仓库自用的订阅转换配置入口 |
| `Clash/kclash.ini` | 兼容旧规则地址的订阅转换配置入口 |
| `Clash/pref.ini` | 规则生成和订阅转换参数配置 |
| `Clash/*.list` | 通用 Clash 规则集 |
| `Clash/Ruleset/*.list` | 按服务拆分的独立规则集 |
| `Providers/*.yml` | Clash rule-provider 格式的规则提供器 |

## 规则顺序原则

Clash 规则按从上到下的顺序匹配，先命中的规则会直接决定策略组。因此本仓库遵循以下顺序原则：

1. 更具体的服务规则放在更通用的规则前面。
2. YouTube、Netflix、Spotify、Discord 等媒体规则应优先于 Google、国外媒体等泛规则。
3. `ProxyMedia` 这类合集规则应放在具体媒体服务之后。
4. AI 服务、开发服务、Web3 服务尽量使用独立规则集，不混入无关规则。
5. DNS 泄漏防护模板中避免加入轻量直连白名单，减少 IP 或 DNS 请求送中的风险。

## 维护约定

- `.list` 文件保持纯文本，一行一条规则。
- 规则格式优先使用 Clash 常见类型：`DOMAIN-SUFFIX`、`DOMAIN-KEYWORD`、`DOMAIN`、`IP-CIDR`、`IP-CIDR6`、`GEOIP`。
- 新增服务时优先创建或修改对应服务规则集，不要把无关服务混在一起。
- 新增规则集后，需要同步检查 `Clash/xiaobai.ini`、`Clash/kclash.ini` 或相关 YAML 模板中的引用。
- 修改规则顺序时，应重点检查是否存在泛规则抢先命中的情况。

## 妙妙屋适配说明

妙妙屋主要使用根目录下的 YAML 模板作为最终配置模板。维护这些模板时，需要优先保证：

- 规则命中顺序正确。
- 媒体、AI、开发、Web3 等服务能命中到对应策略组。
- DNS 相关配置不引入不必要的直连风险。
- 模板中的规则提供器名称与 `rules:` 中引用的名称保持一致。
